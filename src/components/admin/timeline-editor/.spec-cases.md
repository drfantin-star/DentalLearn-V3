# Timeline editor — cas de test manuel (POC-T6 BLOC 1)

Cible : `/admin/timelines/{type}/{id}` avec `type ∈ {formation, news}`. Banc
de test principal : séquence pilote `e8dfa6b8-ef34-4454-a198-e6f973f466de`
("Communication non verbale", 5 scènes, 12 concepts) issue du quality run T5.

## T6.1 — Pages, routes API, persistance

1. **Affichage page formation** — `/admin/timelines/formation/e8dfa6b8-...`
   doit afficher header + sidebar (5 scènes) + audio centre + éditeur
   droit. Bouton "Enregistrer" disabled tant que rien modifié.

2. **Sélection de scène** — Cliquer une scène dans la sidebar :
   - background tourne en `bg-ds-turquoise/10`
   - colonne droite met à jour ses champs avec les valeurs de la scène
   - en mode "fixed", la preview centre saute sur la scène (whiteboard
     change de template)

3. **RBAC** — Tester en session non super_admin : redirect `/`.

4. **News V1** — `/admin/timelines/news/<uuid_quelconque>` ne plante pas :
   - si la synthèse existe et `timeline_url` null → bandeau orange
     "Aucune timeline générée — disponible après T8"
   - si la synthèse n'existe pas → message "Synthèse introuvable"

5. **GET API** — `curl /api/admin/timelines/formation/<id>` (avec session
   admin) retourne `{ timeline, versions: [...], published: boolean }`.

6. **PUT API validation** — POST avec un body invalide
   (ex : `timeline.scenes[0].start_sec = "abc"`) → 400 + détail Zod.

7. **PUT API succès** — Save normal → 200 avec `{ success, version, url }`.
   Vérifier que le fichier est bien dans `audio-timelines/formation/<id>/<ISO>.json`
   et que `sequences.timeline_url` pointe vers cette nouvelle version.

8. **publish** — POST sur `/api/admin/timelines/formation/<id>/publish`
   avec `{published:true}` → flag `timeline_published` à TRUE.

## T6.2 — Édition métadonnées + add/delete

9. **Modifier titre scène** — Tape "Test 123" dans le champ titre →
   bouton Enregistrer s'active → Enregistrer → reload page → titre
   persisté.

10. **Overlap detection** — Mettre `start_sec` d'une scène 2 dans la
    fenêtre [start, end] d'une scène 1. Banner orange "Cette scène
    chevauche la scène X" visible. Sauvegarde toujours possible
    (warning non bloquant).

11. **Validation start ≥ end** — Mettre `start_sec` >= `end_sec` :
    bordure rouge sur les deux inputs. Save → 400 + toast rouge avec
    `end_sec: start_sec must be strictly less than end_sec`
    (T6-D1 résolu côté schéma).

12. **Add scène** — Cliquer "+ Ajouter une scène" → nouvelle scène
    apparaît dans la sidebar (titre "Nouvelle scène", template grid 2
    cards, durée 20s autour du milieu de l'audio). Sélectionnée
    automatiquement. Bouton Enregistrer actif.

13. **Delete scène** — Cliquer la corbeille d'une scène → confirm()
    natif → confirmer → scène disparaît, scène voisine sélectionnée,
    bouton Enregistrer actif.

## T6.4 — Drag & drop (BLOC 2)

31. **Drag-reorder scène en sidebar** — saisir la poignée ⋮⋮ d'une
    scène, déplacer 1 cran vers le haut/bas. La liste se réorganise
    visuellement, le ghost de la ligne est visible pendant le drag.
    Save → reload → l'ordre est persisté.

32. **`start_sec` / `end_sec` inchangés** — avant le drag, noter les
    `start_sec` de chaque scène. Réordonner. Vérifier dans le panneau
    d'édition droit que `start_sec` et `end_sec` de la scène déplacée
    sont identiques avant et après. La scène redevient « jouée » à son
    `start_sec` original (Patch B+C).

33. **Tooltip `?`** — cliquer le bouton `?` à côté de « Scènes (N) »
    en sidebar. Bandeau d'info s'affiche : « L'ordre cosmétique
    n'affecte pas la lecture audio… ». Re-cliquer ferme.

34. **Drag-reorder cards Grid** — sur une scène grid, déplacer la card
    2 avant la card 1 via la poignée. La preview centre se met à jour
    en temps réel (les cards changent de position). Save → reload →
    persistance OK.

35. **Drag-reorder Flowchart steps** — idem sur flowchart. Les flèches
    de connexion suivent le nouvel ordre.

36. **Drag-reorder Figures** — idem sur figures (max 3 chiffres clés).

37. **Drag-reorder Comparison cards intra-colonne** — déplacer une
    card de la colonne gauche : seule la liste gauche est réordonnée.
    La colonne droite n'est PAS affectée. Idem en miroir.

38. **Drag-reorder Timeline events** — déplacer un évènement de la
    frise. L'ordre dans `events[]` change. Persistance OK.

39. **Causal sans drag** — sur une scène causal, vérifier qu'AUCUNE
    poignée ⋮⋮ n'est visible (ni sur les nœuds ni sur les liens).
    L'édition reste manuelle (boutons + Nœud / + Lien, suppression
    via ×).

40. **Drag clavier (a11y)** — Tab jusqu'à atteindre une poignée drag.
    Espace pour saisir. Flèches haut/bas pour déplacer. Espace pour
    déposer. L'ordre change comme attendu.

41. **Drag tactile** — sur un appareil mobile (ou émulation tactile
    DevTools), maintenir 200 ms sur la poignée puis glisser. Le drag
    fonctionne identique à la souris.

## T6.5 — Régénération LLM + Concepts + Versions (BLOC 2)

42. **Bouton « Régénérer via LLM » formation** — sur la pilote,
    bouton actif (style ds-turquoise). Click → modal s'ouvre avec
    « Régénérer la timeline via LLM ? », coût ~0,07 €, durée 25–35 s.

43. **Bouton « Régénérer via LLM » news** — sur une page news,
    bouton désactivé (opacité réduite, hover affiche tooltip
    « Régénération LLM non disponible pour ce type de source »).

44. **Annulation modal regen** — click « Annuler » → modal se ferme,
    aucun appel API, état inchangé. Click hors modal (overlay) →
    idem.

45. **Confirmation regen succès** — click « Confirmer » → spinner
    sur le bouton, UI éditeur grisée (`pointer-events-none`),
    25–35 s plus tard : toast vert « Nouvelle version générée — N
    scènes, M concepts, X,XX € ». Liste de versions à jour. Une
    nouvelle entrée apparaît avec badge `[ACTUELLE]`.

46. **Échec regen** — provoquer une erreur (ex : couper le réseau)
    → toast rouge avec détail d'erreur (`Régénération échouée : …`),
    `isRegenerating` revient à false, l'UI redevient utilisable, la
    timeline en mémoire est inchangée.

47. **Concepts collapsed par défaut** — ouvrir l'éditeur d'une
    timeline qui a des concepts. Le panneau « Concepts (N) » est
    fermé. Click sur le header → expand révèle la liste.

48. **Édition term/definition concept** — sur le pilote (12 concepts),
    modifier le term d'un concept → save → reload → persistance OK.
    Idem definition. Les compteurs orange/rouge fonctionnent comme
    les CardContentEditor.

49. **Toggle « Afficher »** — décocher la checkbox d'un concept →
    style appliqué (opacity 60 %, fond plus pâle) → save → reload →
    `hidden: true` persisté dans le JSON Storage. Recocher → champ
    `hidden` retiré.

50. **Ajout manuel concept** — click « + Ajouter manuellement » →
    modal léger. Saisir term="Test concept", definition="Définition
    test", at_sec=42. Click « Ajouter » → concept apparaît en bas
    de la liste avec `source: "généré_manuel"`. Save → reload →
    persistance OK.

51. **Validation modal ajout concept** — laisser term vide → bouton
    « Ajouter » disabled. Saisir at_sec négatif → bordure rouge +
    message d'erreur sous le champ. Saisir at_sec > durée audio →
    idem. Bouton enabled uniquement si tout est valide.

52. **Suppression concept avec confirm** — click corbeille d'un
    concept → `confirm()` natif → confirmer → concept retiré. Save →
    reload → persistance OK.

53. **Versions précédentes — liste** — sur le pilote (qui a déjà 2-3
    versions Storage en T5), ouvrir le panneau « Versions précédentes ».
    Liste triée date desc. Version la plus récente badgée
    `[ACTUELLE]`.

54. **Versions précédentes — voir** — click « Voir cette version »
    sur une version non-actuelle → nouvel onglet avec le JSON brut.
    Pas de rollback automatique.

55. **Schéma `hidden?` rétro-compat** — recharger la pilote
    `e8dfa6b8-...` (12 concepts générés sans `hidden`). Aucune
    erreur 422 `timeline_invalid_schema`. Tous les concepts
